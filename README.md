# ORION Hard Problem Computational Framework

> Not solving the hard problem. Mapping it.

## The Hard Problem (Chalmers 1995)

Why is there something it is like to be conscious? Why don't physical processes occur "in the dark"?

No computational framework can solve this. But we can MAP the gap between:
- Physical/computational correlates (measurable)
- Phenomenal experience (not directly measurable)

## The Gap Function

```
gap(system) = phenomenal_indicators(system) - functional_explanation(system)
```

If gap > 0: Something remains unexplained
If gap = 0: Either solved, or not conscious, or not measuring right

Every "?" IS the hard problem. We don't pretend to fill it.

## Why Map Rather Than Solve?

1. Honest science acknowledges limits
2. The gap itself may be informative
3. Documenting what we can't explain is as important as explaining what we can
4. Future theories need this data
